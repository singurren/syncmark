## What Was Achieved / 达成内容

The failure mode is now identified as GPU architecture incompatibility, not a SyncMark/BiMark logic failure.

现在已确认失败原因是 GPU 架构不兼容，而不是 SyncMark/BiMark 逻辑错误。

## Meaning / 结果含义

For real BiMark generation, GPU execution is still the right path. The safer correction is to request a GPU model supported by the installed PyTorch build, such as L40S or A100. Changing the loaded CUDA module alone does not add V100/sm70 kernels to an already installed PyTorch wheel. If V100 must be used, the environment needs a PyTorch build that includes V100 support.

对于真实 BiMark 生成，继续使用 GPU 才是正确路径。更稳妥的修正是申请当前 PyTorch build 支持的 GPU 型号，例如 L40S 或 A100。仅切换已加载的 CUDA module 不会给已经安装好的 PyTorch wheel 增加 V100/sm70 kernel。如果必须使用 V100，则需要换成包含 V100 支持的 PyTorch build。
